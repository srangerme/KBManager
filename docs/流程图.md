# API 流程图

本文只描述第二层 `kb.*` API 流程。第一层交互与编排边界见 `docs/Interface.md`。

标记含义：

- `(api)`：第二层 Application API / Domain Service。
- `(data)`：第三层 Repository / Data Layer。
- `(LLM)`：由 Claude Code 接管的 LLM 调用。API 只返回 `needs_llm` 和 `llm_request`，不直接调用 LLM；用户侧不提供提示词文件。
- `(hook)`：Claude Code PreToolUse hook 在最终写入 helper 调用执行前请求用户审批。



## 流程影响审查

| API | 结论 | 说明 |
| --- | --- | --- |
| `kb.init` | 不改主流程 | 初始化目录和模板占位，不涉及新知识模型的运行流程。 |
| `kb.source.add` | 修改 | source ingest 只创建独立 source，初始状态为 `raw`。 |
| `kb.source.deprecate` | 不改主流程 | 废弃 source 的 review gate 不受模型变化影响。 |
| `kb.candidate.create` | 修改 | 输入为已有 source IDs；需要读取 knowledgebase `description/scope/outlines`，生成 `bindto` 和 outline 修改建议，并同步 source `linked` 状态。 |
| `kb.candidate.get` | 字段同步 | 读取流程不变，返回内容包含 `summary/content/bindto/outline_change_suggestions`。 |
| `kb.candidate.next_pending` | 不改主流程 | 排队逻辑不受字段变化影响。 |
| `kb.candidate.defer` | 不改主流程 | 状态转换由 PreToolUse hook 审批后执行。 |
| `kb.knowledge.accept` | 修改 | reviewed payload 改为 `summary/content/evidence/bindto`，不修改 outline。 |
| `kb.knowledge.merge` | 修改 | 合并 payload 改为 `summary/content/evidence/bindto`，不修改 outline。 |
| `kb.knowledge.reject` | 不改主流程 | 拒绝 candidate 由 PreToolUse hook 审批后执行，不依赖新字段。 |
| `kb.knowledge.deprecate` | 不改主流程 | 废弃 knowledge 的状态流转不变。 |
| `kb.knowledgebase.create` | 修改 | 草案由 `kb.knowledgebase.create.prepare` / `revise` 生成；最终 API 接收经用户 review 的完整 payload 并由 PreToolUse hook 审批后写入。 |
| `kb.knowledgebase.outline.create` | 新增 | 追加经 review 的新 outline，并更新 manifest。 |
| `kb.knowledgebase.outline.set_default` | 新增 | 切换 active knowledgebase 的默认 outline。 |
| `kb.knowledgebase.outline.archive` | 新增 | 归档非默认 outline，并检查已有绑定影响。 |
| `kb.knowledgebase.map` | 修改 | 使用默认 outline + `bindto` 结构图。 |
| `kb.note.add` | 不改主流程 | note 写入流程不受知识模型变化影响。 |
| `kb.note.get` | 不改主流程 | note 读取流程不受知识模型变化影响。 |
| `kb.note.deprecate` | 不改主流程 | note 废弃流程由 PreToolUse hook 审批后执行。 |
| `kb.index.rebuild` | 修改 | 增加 `bindto`、outline 节点和 source `raw/linked` 状态一致性检查。 |

## 1. `kb.init`

```mermaid
flowchart TD
  A["(api) 接收目标目录，默认当前调用目录"] --> B["(api) 校验目录可写且不越界"]
  B --> C["(data) 读取默认目录清单和模板"]
  C --> D["(data) 扫描已存在路径和冲突"]
  D --> E{"(api) 是否存在覆盖风险或不兼容结构"}
  E -- 是 --> F["(api) 返回 failed 和冲突原因，不写入"]
  G -- 是 --> H["(api) 返回拟创建路径和冲突列表"]
  G -- 否 --> I["(data) 创建受控目录、模板和索引占位"]
  I --> J["(api) 返回创建列表、已存在列表和下一步建议"]
```

## 2. `kb.source.add`

```mermaid
flowchart TD
  A["(api) 接收单个文件"] --> B["(api) 校验输入类型"]
  B --> D["(data) 读取本地文件输入"]
  D --> M["(data) 读取 source 模板"]
  M --> N["(api) 返回 needs_llm: source ingest"]
  N --> O["(LLM) Claude Code 按 llm_request 生成 summary 和 tags"]
  O --> P["(api) resume 并校验 summary 和 tags"]
  P --> Q["(api) 分配 source ID 并校验唯一性"]
  Q --> R["(data) 写入 raw source 对象和原始资源引用"]
  R --> S["(api) 返回 source_ids 和 source summary"]
```

## 3. `kb.source.deprecate`

```mermaid
flowchart TD
  A["(api) 接收 source ID 和原因"] --> B["(data) 读取 source 和引用关系"]
  B --> C["(api) 计算引用影响列表"]
  C --> D["(hook) PreToolUse 请求用户审批最终写入 helper 调用"]
  D --> F["(data) 写入 deprecated 状态和原因"]
  F --> G["(api) 返回 deprecated source、影响列表和自动索引重建结果"]
```

## 4. `kb.candidate.create`

```mermaid
flowchart TD
  A["(api) 接收 source_ids 引用"] --> B["(data) 读取上游 source、candidate 模板和 knowledgebase 摘要"]
  B --> C["(data) 读取 active knowledgebase 的 description、scope、default_outline_id 和 outlines"]
  C --> D["(api) 返回 needs_llm: candidate draft list + bindto/outline suggestions"]
  D --> E["(LLM) Claude Code 按 llm_request 和 knowledgebase 定义判断 source 可提取知识"]
  E --> F["(LLM) 按 schema 生成 title、summary、content、evidence、bindto 和 outline_change_suggestions"]
  F --> G["(api) resume 并校验 title、summary、content、evidence、bindto 和全局 knowledge ID"]
  G --> H["(data) 写入 pending candidate Markdown"]
  H --> S["(data) 同步被有效引用 source 的 linked 状态"]
  S --> I["(api) 返回 candidate/knowledge ID、bindto 和 outline 修改建议"]
```

## 7. `kb.candidate.defer`

```mermaid
flowchart TD
  A["(api) 接收 candidate/knowledge ID 和延后备注"] --> B["(data) 读取 candidate"]
  B --> C["(api) 校验状态转换和 reason"]
  C --> H["(hook) PreToolUse 请求用户审批最终写入 helper 调用"]
  H --> D["(data) 写入 deferred 状态、reviewed_at、decision 和 reason"]
  D --> E["(api) 返回 deferred candidate 和自动索引重建结果"]
```

## 8. `kb.knowledge.accept`

```mermaid
flowchart TD
  A["(api) 接收 candidate/knowledge ID、accept 决策和用户 review 后内容"] --> B["(data) 读取 candidate 和 knowledge 模板"]
  B --> C["(data) 读取 bindto 指向的 knowledgebase outline"]
  C --> D["(api) 校验 reviewed payload: evidence、summary、content、bindto 和同 ID规则"]
  D --> G["(hook) PreToolUse 请求用户审批最终写入 helper 调用"]
  G --> H["(api) 按模板将 candidate 转换为同 ID knowledge 对象"]
  H --> I["(data) 原子提升/迁移 candidate 文件到 knowledge/atomic"]
  I --> J["(data) 更新 type/status、summary/content/evidence/bindto 和 review 字段，不修改 knowledgebase outline 或成员列表"]
  J --> K["(api) 返回 knowledge ID、bindto 和自动索引重建结果"]
```

## 9. `kb.knowledge.merge`

```mermaid
flowchart TD
  A["(api) 接收 pending candidate、目标 knowledge、merge 决策和用户 review 后内容"] --> B["(data) 读取 candidate、目标 knowledge、来源和 knowledgebase 摘要"]
  B --> C["(data) 读取 bindto 指向的 knowledgebase outline"]
  C --> D["(api) 校验 reviewed payload: evidence、summary、content、bindto 和目标 knowledge"]
  D --> G["(hook) PreToolUse 请求用户审批最终写入 helper 调用"]
  G --> H["(data) 用目标 knowledge ID 写入合并后的 knowledge"]
  H --> I["(data) 更新目标 knowledge 的 summary/content/evidence/bindto 和 review 字段，来源 candidate rejected，不修改 knowledgebase outline 或成员列表"]
  I --> J["(api) 返回合并结果、bindto 和自动索引重建结果"]
```

## 12. `kb.knowledgebase.create`

```mermaid
flowchart TD
  A["(api) kb.knowledgebase.create.prepare 接收 title 和临时 input_path"] --> B["(data) 读取 source-like context、knowledgebase 模板和已有 knowledgebase 摘要"]
  B --> C["(api) 返回 needs_llm: knowledgebase draft"]
  C --> D["(LLM) Claude Code 按 llm_request 生成 description、tags、scope、default_outline_id 和 outlines"]
  D --> E["(api) resume prepare 并校验 draft，返回 reviewed_payload，不写入"]
  E --> F["(user) approve 或提交 review note"]
  F -- note --> R["(api) kb.knowledgebase.create.revise 返回 revised payload，不写入"]
  R --> F
  F -- approve --> G["(hook) PreToolUse 请求用户审批最终写入 helper 调用"]
  G --> H["(api) kb.knowledgebase.create 校验 title、ID、description、scope、default_outline_id 和 outlines"]
  H --> I["(data) 写入 active knowledgebase Markdown"]
  I --> J["(data) 写入完整 outlines YAML"]
  J --> K["(api) 返回 knowledgebase ID、Markdown 路径、outlines YAML 路径和自动索引重建结果"]
```

## 15. `kb.knowledgebase.outline.archive`

```mermaid
flowchart TD
  A["(api) 接收 knowledgebase_id、outline_id 和 allow_existing_bindings"] --> B["(data) 读取 active knowledgebase、outlines YAML 和 knowledge bindto"]
  B --> C["(hook) PreToolUse 请求用户审批最终写入 helper 调用"]

  F -- 否 --> H["(data) 标记 outline archived 并更新 manifest"]
```
