# API 流程图

本文只描述第二层 `kb.*` API 流程。Slash command 的交互流程见 `docs/SlashCommand流程图.md`。

标记含义：

- `(api)`：第二层 Application API / Domain Service。
- `(data)`：第三层 Repository / Data Layer。
- `(LLM)`：由 Claude Code 接管的 LLM 调用。API 只返回 `needs_llm` 和 `llm_request`，不直接调用 LLM；用户侧不提供提示词文件。
- `(user)`：API 需要 user review 或确认时的外部决策输入。

## 1. `kb.init`

```mermaid
flowchart TD
  A["(api) 接收目标目录，默认当前调用目录"] --> B["(api) 校验目录可写且不越界"]
  B --> C["(data) 读取默认目录清单和模板"]
  C --> D["(data) 扫描已存在路径和冲突"]
  D --> E{"(api) 是否存在覆盖风险或不兼容结构"}
  E -- 是 --> F["(api) 返回 failed 和冲突原因，不写入"]
  E -- 否 --> G{"(api) 是否 dry_run"}
  G -- 是 --> H["(api) 返回拟创建路径和冲突列表"]
  G -- 否 --> I["(data) 创建受控目录、模板和索引占位"]
  I --> J["(api) 返回创建列表、已存在列表和下一步建议"]
```

## 2. `kb.source.add`

```mermaid
flowchart TD
  A["(api) 接收目录/文件/URL"] --> B["(api) 解析输入类型"]
  B --> C["(data) 读取 source 模板"]
  C --> D["(api) 返回 needs_llm: source ingest"]
  D --> E["(LLM) Claude Code 生成 summary、cleaned_content、metadata_suggestions"]
  E --> F["(api) resume 并校验 summary 和 cleaned_content"]
  F --> G["(api) 分配 source ID 并校验唯一性"]
  G --> H["(data) 写入 source 对象、cleaned 派生字段和原始资源引用"]
  H --> I["(api) 返回 source ID、总结和 cleaned 派生字段引用"]
```

## 3. `kb.source.deprecate`

```mermaid
flowchart TD
  A["(api) 接收 source ID 和原因"] --> B["(data) 读取 source 和引用关系"]
  B --> C["(api) 计算引用影响列表"]
  C --> D{"(api) 是否已有 user 确认"}
  D -- 否 --> E["(api) 返回 needs_review"]
  D -- 是 --> F["(data) 写入 deprecated 状态和原因"]
  F --> G["(api) 返回 deprecated source、影响列表和自动索引重建结果"]
```

## 4. `kb.candidate.create`

```mermaid
flowchart TD
  A["(api) 接收 source_ids/note_ids 引用"] --> B["(data) 读取上游对象、candidate 模板和 knowledgebase 摘要"]
  B --> C["(api) 返回 needs_llm: candidate draft list + tag/kb suggestions"]
  C --> D["(LLM) Claude Code 生成 candidate draft list、tag 建议和 knowledgebase 建议"]
  D --> E["(api) resume 并校验来源、证据和全局 knowledge ID"]
  E --> F["(data) 写入 pending candidate Markdown"]
  F --> G["(api) 返回 candidate/knowledge ID、tag 建议和 knowledgebase 建议"]
```

## 5. `kb.candidate.get`

```mermaid
flowchart TD
  A["(api) 接收 candidate/knowledge ID"] --> B["(data) 读取 candidate Markdown"]
  B --> C["(data) 读取必要引用摘要"]
  C --> D["(api) 返回路径、frontmatter、正文摘要"]
```

## 6. `kb.candidate.next_pending`

```mermaid
flowchart TD
  A["(api) 接收可选过滤条件"] --> B["(data) 读取 pending candidate 或 review queue"]
  B --> C["(api) 按添加时间排序"]
  C --> D["(api) 返回下一个 pending candidate"]
```

## 7. `kb.candidate.defer`

```mermaid
flowchart TD
  A["(api) 接收 candidate/knowledge ID 和延后备注"] --> B["(data) 读取 candidate"]
  B --> C["(api) 校验 user defer 决策和状态转换"]
  C --> D["(data) 写入 deferred 状态和 review 记录"]
  D --> E["(api) 返回 deferred candidate 和自动索引重建结果"]
```

## 8. `kb.knowledge.accept`

```mermaid
flowchart TD
  A["(api) 接收 candidate/knowledge ID、accept 决策和用户 review 后内容"] --> B["(data) 读取 candidate 和 knowledge 模板"]
  B --> C["(api) 校验 user accept 决策、证据、tags、kb_ids 和同 ID 规则"]
  C --> H["(api) 按模板将 candidate 转换为同 ID knowledge 对象"]
  H --> I["(data) 原子提升/迁移 candidate 文件到 knowledge/atomic"]
  I --> J["(data) 更新 type/status、review 字段、knowledge kb_ids 和 knowledgebase knowledge_ids，不保留同 ID candidate"]
  J --> K["(api) 返回 knowledge ID、kb_ids 和自动索引重建结果"]
```

## 9. `kb.knowledge.merge`

```mermaid
flowchart TD
  A["(api) 接收 pending candidate、目标 knowledge、merge 决策和用户 review 后内容"] --> B["(data) 读取 candidate、目标 knowledge、来源、关系和 knowledgebase 摘要"]
  B --> C["(api) 校验 user merge 决策、review 后内容和目标 knowledge"]
  C --> H["(data) 用目标 knowledge ID 写入合并后的 knowledge"]
  H --> I["(data) 更新目标 knowledge，来源 candidate rejected，更新 kb_ids 和 knowledgebase knowledge_ids"]
  I --> J["(api) 返回合并结果、kb_ids 和自动索引重建结果"]
```

## 13. `kb.note.add`

```mermaid
flowchart TD
  A["(api) 接收 note Markdown 内容"] --> B["(data) 读取 note 模板"]
  B --> C{"(api) 是否需要 LLM 标题摘要"}
  C -- 是 --> D["(api) 返回 needs_llm: note title summary"]
  D --> E["(LLM) Claude Code 生成标题和摘要"]
  E --> F["(api) resume 并校验结果"]
  C -- 否 --> F
  F --> G["(api) 分配 note ID"]
  G --> H["(data) 写入 note Markdown"]
  H --> I["(api) 返回 note ID 和路径"]
```

## 16. `kb.index.rebuild`

```mermaid
flowchart TD
  A["(api) 接收重建范围和 dry_run 选项"] --> B["(data) 扫描对象文件、frontmatter、meta 和引用关系"]
  B --> C["(api) 基于对象事实生成目标索引内容"]
  C --> D{"(api) 是否 dry_run"}
  D -- 是 --> E["(api) 返回拟更新 diff 和一致性问题"]
  D -- 否 --> F["(data) 覆盖写入派生索引文件"]
  F --> G["(api) 返回重建索引路径、diff 和未修复项"]
```
